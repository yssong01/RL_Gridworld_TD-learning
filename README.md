# RL Gridworld TD Learning

4x4 GridWorld 환경에서 TD Learning과 Monte Carlo Learning을 구현하고, 일반 환경과 무작위 벽 환경 간의 상태가치 학습 결과를 비교 분석한다.

---

## 1. 연구 목적

본 프로젝트는 강화학습의 대표적 알고리즘인 시간차 학습(Temporal Difference Learning)과 몬테카를로 학습(Monte Carlo Learning)을 동일 환경에서 구현하고, 두 가지 환경 조건(일반 환경 / 무작위 벽 환경)에서 학습된 상태가치 함수 V(s)의 수렴 특성을 정량적으로 비교하는 것을 목적으로 한다.

---

## 2. 환경 구성

### 2.1 공통 설정

| 항목 | 설정값 |
|---|---|
| 격자 크기 | 4 x 4 |
| 시작 상태 | (0, 0) — 좌상단 |
| 목표 상태 | (3, 3) — 우하단 |
| 행동 공간 | 상, 하, 좌, 우 (4방향) |
| 정책 | 균등 무작위 정책 (각 25%) |
| 보상 | 매 스텝 −1 (목표 도달 시 에피소드 종료) |
| 경계 및 벽 충돌 | 이전 위치 유지 (−1 보상은 계속 누적) |

### 2.2 Standard GridWorld

장애물이 없는 기본 4x4 격자 환경이다. 에이전트는 경계 조건만 고려하여 자유롭게 이동할 수 있다.

### 2.3 Random Wall GridWorld

매 에피소드 시작 시 2개 또는 3개의 벽이 격자 내 무작위 위치에 생성된다. 시작 지점 (0, 0)과 목표 지점 (3, 3)에는 벽이 생성되지 않으며, 같은 위치에 중복 생성되지 않는다. 벽의 위치는 에피소드마다 독립적으로 재결정된다.

---

## 3. 알고리즘

### 3.1 TD(0) Learning

에피소드 종료를 기다리지 않고, 매 스텝 이동 직후 다음 상태의 가치 추정치를 이용하여 현재 상태의 가치를 즉시 갱신한다.

```
V(s) <- V(s) + alpha * [R + gamma * V(s') - V(s)]
```

- `alpha` : 학습률 (Learning Rate)
- `R` : 현재 스텝에서 받은 보상
- `gamma` : 할인율 (Discount Factor)
- `V(s')` : 다음 상태의 현재 가치 추정치

### 3.2 Monte Carlo Learning

에피소드가 완전히 종료된 후, 해당 에피소드의 전체 누적 보상 G를 역방향으로 계산하여 방문한 모든 상태의 가치를 일괄 갱신한다.

```
V(s) <- V(s) + alpha * [G - V(s)]
```

- `G` : 에피소드 종료 시점부터 역방향으로 계산한 누적 할인 보상

### 3.3 하이퍼파라미터

| 파라미터 | 값 |
|---|---|
| 에피소드 수 | 50,000 |
| 학습률 alpha | 0.01 |
| 할인율 gamma | 1.0 |
| 기록 간격 | 500 에피소드마다 V(0,0) 기록 |
| 최대 스텝 (TD) | 에피소드당 1,000 스텝 |
| 최대 스텝 (MC) | 에피소드당 500 스텝 |

## 4. 실험 결과

### 4.1 상태가치 히트맵

TD Learning으로 학습된 V(s) 테이블을 히트맵으로 시각화한다. 색상은 파랑(가장 낮은 가치)에서 붉은색(목표=0)으로 부드럽게 전환되며, 격자는 정사각형으로 출력된다.

**Standard GridWorld 결과 예시 (V(s) 테이블)**

```
       X=0      X=1      X=2      X=3
Y=0   -59.43   -57.45   -54.71   -51.52
Y=1   -57.50   -54.53   -50.30   -44.93
Y=2   -54.55   -50.35   -41.91   -31.48
Y=3   -51.69   -44.58   -30.72     0.00
```

**Random Wall GridWorld 결과 예시 (V(s) 테이블)**

```
       X=0      X=1      X=2      X=3
Y=0  -145.66  -139.71  -129.17  -121.63
Y=1  -143.08  -132.19  -120.28  -112.68
Y=2  -129.81  -122.01  -104.33   -65.02
Y=3  -123.86  -114.05   -65.68     0.00
```

### 4.2 환경 간 비교 (TD Learning 기준)

| 상태 | Standard TD | Random Wall TD | 차이 |
|---|---|---|---|
| V(0,0) — 시작점 | -59.43 | -145.66 | -86.22 |
| V(3,0) — 우상단 | -51.52 | -121.63 | -70.12 |
| V(0,3) — 좌하단 | -51.69 | -123.86 | -72.16 |
| V(2,2) — 목표 근처 | -41.91 | -104.33 | -62.42 |
| V(3,3) — 목표점 | 0.00 | 0.00 | 0.00 |

무작위 벽 환경의 V(0,0)은 일반 환경 대비 약 2.45배 더 큰 음수로 수렴하였다.

### 4.3 결과 해석

무작위 벽 환경에서 상태가치가 크게 낮아지는 원인은 세 가지로 분석된다.

1. **불필요한 스텝 누적**: 벽 방향으로 이동을 시도하면 위치는 유지되지만 −1 보상이 계속 누적된다. 유효한 이동 없이 페널티만 반복 부과된다.

2. **최단 경로 차단**: 에피소드마다 다르게 생성되는 벽이 기존의 지름길을 막아, 에이전트는 매번 우회 경로를 탐색하게 된다. 이로 인해 목표 도달까지의 기대 스텝 수가 증가한다.

3. **갇힘(Trapped) 현상**: 드물지만 에이전트 주변이 벽으로 둘러싸이는 경우, 장기간 탈출이 불가능하여 해당 에피소드에서 매우 큰 음수 보상이 누적된다. 이 경우가 전체 평균 가치를 크게 끌어내린다.

### 4.4 TD vs Monte Carlo 비교

TD Learning은 에피소드 종료 전에도 매 스텝마다 점진적으로 업데이트가 이루어진다. 이로 인해 특히 에피소드가 길어지는 무작위 벽 환경에서 Monte Carlo 대비 더 안정적이고 빠른 수렴을 보인다. Monte Carlo는 완성된 에피소드의 실제 누적 보상을 사용하므로 편향이 없는 반면, 에피소드 길이에 따른 분산이 커 수렴 곡선의 진폭이 상대적으로 크다.

## 5. 파일 구조

```
RL_Gridworld_TD-learning/
│
├── doit.ipynb                   # 전체 구현 및 실험 Jupyter Notebook
│
└── Results/                     # 코드 실행 시 자동 생성되는 결과 폴더
    ├── learning_history.csv         # 에피소드별 V(0,0) 학습 이력 (4가지 조합)
    ├── heatmap_standard.png         # Standard GridWorld V(s) 히트맵
    ├── heatmap_random_wall.png      # Random Wall GridWorld V(s) 히트맵
    ├── heatmap_comparison.png       # 두 환경 히트맵 비교 (나란히)

```

`doit.ipynb` 내 셀 구성:

| 셀 | 내용 |
|---|---|
| 셀 1 | 라이브러리 임포트, GridWorld 클래스 정의 (Standard / RandomWall), Agent 클래스 |
| 셀 2 | `train_td()` / `train_mc()` 함수 정의 및 4가지 조합 학습 실행, CSV 저장 |
| 셀 3 | V(s) 히트맵 시각화 (개별 2장 + 비교 1장) |
| 셀 4 | 학습 곡선 시각화 (3개 그래프) |
| 셀 5 | 수치 비교표 및 결과 분석 출력 |
